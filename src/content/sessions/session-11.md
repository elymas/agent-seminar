## 세션 개요

이 세션은 비전 세션입니다. 6주간의 여정이 궁극적으로 어디를 향하는지 보여주는 자리입니다.

지난 5주 동안 우리는 프롬프트 작성부터 시작해 컨텍스트 설계, Skill 제작, MCP 연동, 가드레일 설정까지 차근차근 쌓아올렸습니다. 이 모든 것의 끝에는 하나의 질문이 있습니다. **"AI가 사람 없이도 일할 수 있을까?"** 이번 세션에서 그 답을 살펴봅니다.

## 학습 목표

- Scheduled Tasks, Remote Control, Channels의 개념과 역할을 각각 설명할 수 있다
- 세 가지 도구가 결합되었을 때 어떤 업무 시나리오가 가능해지는지 구체적으로 그려볼 수 있다
- 6주간의 Adoption Ladder 전체 여정을 돌아보며, 자신이 어디까지 왔는지 인식할 수 있다
- Season 2에서 다루고 싶은 주제를 구체적으로 제안할 수 있다

## 6주간의 여정 돌아보기

먼저 우리가 어디까지 왔는지 전체 그림을 한눈에 봅시다.

Week 1에서 AI에게 첫 질문을 던지던 때를 기억하시나요? 지금 우리는 AI가 스스로 판단하고, 규칙을 따르고, 외부 도구까지 사용하는 단계까지 왔습니다. 오늘은 이 여정의 마지막 퍼즐 조각을 맞춥니다. AI가 **사람이 없는 시간에도** 일하게 하는 것.

## Level 5: Operating Layer

Level 5는 AI 에이전트가 사람의 직접적인 지시 없이도 **자율적으로 작동**하는 단계입니다. 이를 가능하게 하는 세 가지 핵심 도구가 있습니다.

### Scheduled Tasks (예약 작업)

정해진 시간에 AI가 자동으로 작업을 실행합니다. 사람이 매번 지시하지 않아도, 미리 정해둔 일정에 따라 AI가 스스로 움직입니다.

**작동 원리:**
- 시간 조건(매일 오전 8시, 매주 월요일 등)을 설정
- 조건이 충족되면 AI가 지정된 Skill을 자동 실행
- 결과물을 지정된 채널(Slack, 이메일 등)로 전달

**광고 미디어렙에서의 활용 예시:**

| 일정 | 자동 실행 작업 | 결과물 |
|------|--------------|--------|
| 매일 오전 8시 | 전일 미디어 성과 데이터 수집 → 요약 브리프 생성 | Slack에 일일 성과 요약 자동 발행 |
| 매주 월요일 오전 9시 | 경쟁사 광고 동향 분석 | 주간 경쟁사 브리프 자동 생성 |
| 매월 1일 | 월간 미디어 성과 데이터 종합 | 월간 성과 보고서 초안 자동 생성 |

이것이 가능한 이유는 우리가 이미 기반을 쌓았기 때문입니다. Week 2에서 만든 컨텍스트가 AI에게 "어떤 톤으로 작성할지"를 알려주고, Week 3에서 만든 Skill이 "어떤 순서로 작업할지"를 정의하고, Week 4에서 설계한 MCP 연동이 "어디서 데이터를 가져올지"를 연결합니다. Scheduled Tasks는 이 모든 것 위에 **"언제 실행할지"**만 추가하는 것입니다.

### Remote Control (원격 이어받기)

이미 실행 중인 AI 세션을 다른 기기에서 이어받을 수 있습니다. PC에서 시작한 작업을 모바일에서, 사무실에서 시작한 작업을 집에서 계속할 수 있습니다.

**작동 원리:**
- AI 세션은 클라우드에서 지속적으로 실행
- 사용자는 어떤 기기에서든 해당 세션에 접속 가능
- 진행 상황 확인, 추가 지시, 승인/반려 모두 원격으로 처리

**실무 시나리오:**

1. **오후 3시, 사무실 PC에서** -- AI에게 "이번 달 A뷰티 캠페인 성과 분석 보고서를 작성해줘"라고 지시
2. **오후 4시, 클라이언트 미팅 중** -- 모바일로 AI 세션에 접속하여 진행 상황 확인. "분석 범위에 TikTok도 포함해줘"라고 추가 지시
3. **오후 6시, 퇴근 후** -- 모바일에서 완성된 보고서 확인. "좋아, 이대로 팀 Slack에 공유해줘"라고 승인

핵심은 AI의 작업이 **내 자리에 앉아 있을 때만** 가능한 것이 아니라는 점입니다. AI는 계속 일하고 있고, 사람은 필요한 순간에만 개입하면 됩니다.

### Channels (외부 이벤트 통로)

외부에서 발생하는 이벤트(Slack 메시지, 이메일, 알림 등)를 AI에게 자동으로 전달합니다. AI가 먼저 상황을 감지하고, 사람에게 보고하거나 미리 준비 작업을 시작합니다.

**작동 원리:**
- 외부 이벤트 소스(Slack, 이메일, 모니터링 시스템 등)를 Channel로 연결
- 특정 조건에 맞는 이벤트가 발생하면 AI가 자동으로 수신
- AI가 이벤트를 분석하고, 미리 정해둔 규칙에 따라 행동

**실무 시나리오:**

1. **클라이언트가 Slack으로 긴급 요청을 보냄** -- "내일 미팅 전에 최근 2주 성과 데이터가 필요합니다"
2. **Channel이 이 메시지를 AI에게 전달** -- AI가 메시지 내용을 분석하고 "긴급 데이터 요청"으로 분류
3. **AI가 자동으로 관련 데이터 수집 시작** -- 해당 클라이언트의 최근 2주 성과 데이터를 수집하고 요약 초안 생성
4. **담당 AE에게 알림** -- "A클라이언트 긴급 데이터 요청이 있었습니다. 초안을 준비해뒀으니 확인 후 전달해주세요"

사람이 메시지를 확인하기도 전에, AI가 이미 준비 작업을 시작합니다. 담당자는 0에서 시작하는 대신, AI가 만든 80% 완성된 초안 위에서 마무리만 하면 됩니다.

## 비전: AI가 24/7 돌아가는 광고 운영

세 가지 도구가 결합되면 어떤 하루가 가능해질까요? 미래의 한 장면을 그려봅시다.

```
06:00  Scheduled Task 실행
       → 전일 매체별 성과 데이터 자동 수집
       → 이상치 감지 (CPM 30% 급등)
       → Slack 알림: "A뷰티 Meta 캠페인 CPM 이상 감지"

09:00  출근
       → AI가 정리한 오늘의 우선순위 확인
       → "A뷰티 CPM 이상 → 타겟팅 조정 필요"

10:30  클라이언트 미팅 중
       → Remote Control로 AI 세션 이어받기
       → "이 데이터 기반으로 대안 3가지 준비해줘"

14:00  Channel 알림
       → 경쟁사 B브랜드 대규모 캠페인 런칭 감지
       → AI가 자동으로 비교 분석 브리프 생성

16:00  퇴근 준비
       → AI가 오늘의 작업 요약 + 내일 To-Do 자동 정리
       → 내일 Scheduled Task에 반영
```

출근 전에 AI가 이미 데이터를 수집하고 이상 징후를 보고합니다. 미팅 중에도 모바일로 AI에게 추가 지시를 내립니다. 경쟁사 동향이 감지되면 AI가 먼저 분석을 시작합니다. 퇴근 후에도 AI는 다음날을 준비합니다.

이것은 먼 미래의 이야기가 아닙니다. 우리가 6주간 쌓아온 모든 것 -- 프롬프트, 컨텍스트, Skill, MCP, 가드레일 -- 이 이 비전의 기반입니다.

**중요한 원칙:** Level 5에서도 **사람은 여전히 의사결정의 중심**입니다. AI가 자율적으로 데이터를 수집하고 초안을 만들지만, 최종 판단과 승인은 항상 사람의 몫입니다. Week 5에서 배운 가드레일이 이 단계에서 더욱 중요해집니다.
