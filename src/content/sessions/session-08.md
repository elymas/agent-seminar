### 세션 개요

세션 7에서 MCP와 Connector의 개념을 배웠습니다. AI가 외부 도구를 직접 사용할 수 있다는 것을 알게 되었으니, 이번 세션에서는 **자신의 실제 업무**에 이것을 어떻게 적용할 수 있는지 설계합니다.

중요한 점: 이번 세션은 **설계** 실습입니다. 실제로 MCP를 연결하거나 자동화를 구축하지 않습니다. "설계도"를 그리는 단계입니다. 실제 구현은 Season 2에서 다룹니다.

### 학습 목표

- 자신의 업무에서 수동 도구 이동이 발생하는 지점을 식별할 수 있다
- AS-IS(현재 수동 프로세스)와 TO-BE(자동화 후 프로세스)를 구조화하여 비교할 수 있다
- 자동화 시나리오의 실현 가능성을 4가지 기준으로 평가할 수 있다

---

### 미션 브리핑 (5분)

**미션**: 자신의 업무에서 외부 도구 연동이 필요한 자동화 시나리오 1개를 **설계**하세요.

이번 미션의 핵심은 "지금 바로 만들 수 있는가?"가 아니라, **"무엇을 자동화하면 가장 효과적인가?"**를 판단하는 것입니다. 좋은 설계도가 있어야 좋은 건물이 지어지듯, 자동화도 설계가 먼저입니다.

**진행 순서:**
1. 도구 이동 맵 그리기 (7분) -- 현재 수동으로 도구를 옮겨 다니는 작업을 시각화
2. 자동화 시나리오 설계 (8분) -- AS-IS → TO-BE로 변환
3. 실현 가능성 평가 (5분) -- 이 시나리오가 실제로 구현 가능한지 체크

---

### 실습 가이드 (20분)

#### Step 1: 도구 이동 맵 그리기 (7분)

먼저 현재 업무에서 **수동으로 도구를 옮겨 다니는 작업**을 찾아봅니다. "이 도구에서 저 도구로 데이터를 손으로 옮기는" 모든 작업이 해당됩니다. 아래 템플릿을 채워보세요.

```markdown
## 나의 도구 이동 맵

현재 업무에서 수동으로 도구를 옮겨 다니는 작업을 찾아보세요.

**업무명**: [여기에 입력]

### 현재 흐름 (수동)
[도구 A] → 수동 복사 → [도구 B] → 수동 정리 → [도구 C]
예: GA 대시보드 → 스크린샷 → Excel → 수동 정리 → PPT → 이메일 발송

### 시간 소요
- 도구 A에서 데이터 가져오기: [X]분
- 도구 B에서 정리: [X]분
- 도구 C로 전달: [X]분
- 총 소요: [X]분 x [빈도] = 주 [X]시간
```

> **Tip**: 작업 자체가 어려운 것이 아니라 **도구 사이를 왔다 갔다 하느라** 시간이 걸리는 업무가 자동화 효과가 가장 큽니다. "이건 AI가 할 수 있는 일인가?"보다 "이건 사람이 하기엔 아까운 단순 반복인가?"를 기준으로 찾아보세요.

#### Step 2: 자동화 시나리오 설계 (8분)

Step 1에서 찾은 수동 프로세스를 자동화 시나리오로 변환합니다. 아래 템플릿의 각 항목을 채워보세요.

```markdown
## 자동화 시나리오

### AS-IS (현재)
[수동 프로세스를 그대로 기술]

### TO-BE (자동화 후)
입력: [어디서 데이터가 시작되는가?]
  ↓ [어떤 MCP/도구로 연결?]
처리: [AI가 무엇을 하는가?]
  ↓ [어떤 MCP/도구로 연결?]
출력: [결과가 어디로 가는가?]

### 필요한 연동
1. [도구명] — [연결 목적]
2. [도구명] — [연결 목적]

### 기대 효과
- 절약 시간: 주 [X]시간 → [Y]시간 (절감: [Z]시간)
- 품질 향상: [구체적으로 어떤 점이 좋아지는가?]
- 추가 가치: [예: 실시간 모니터링 가능]
```

**작성 예시: 주간 미디어 성과 보고서 자동화**

아래는 "주간 미디어 성과 보고서"를 자동화하는 시나리오의 완성 예시입니다. 이 수준으로 작성해보세요.

```markdown
## 자동화 시나리오

### AS-IS (현재)
1. 매주 월요일 오전, GA4에 접속하여 지난주 캠페인 성과를 확인 (10분)
2. 매체별(Meta, Google, 네이버) 관리자 화면에서 데이터를 각각 복사 (15분)
3. Excel에 붙여넣고 전주 대비 증감률을 수동 계산 (15분)
4. PPT 템플릿에 표와 차트를 옮겨 작성 (20분)
5. 완성된 보고서를 Slack에 업로드하고 팀장에게 멘션 (5분)
총 소요: 약 65분 x 매주 = 주 1시간+

### TO-BE (자동화 후)
입력: Google Sheets (매체별 성과 데이터가 자동 집계되는 시트)
  ↓ Google Sheets Connector
처리: AI가 전주 대비 증감률 계산, 매체별 성과 테이블 정리,
      상위 3개 인사이트 도출, 보고서 형식으로 작성
  ↓ Slack Connector
출력: 미디어팀 Slack 채널에 주간 보고 요약 자동 발행

### 필요한 연동
1. Google Sheets — 매체별 성과 데이터 읽기
2. Slack — 완성된 보고서를 채널에 발행

### 기대 효과
- 절약 시간: 주 65분 → 5분 (절감: 60분/주, 연간 약 50시간)
- 품질 향상: 수동 계산 오류 제거, 매주 동일한 형식 보장
- 추가 가치: 월요일 오전 9시 자동 발행으로 팀 전체가 빠르게 현황 파악
```

#### Step 3: 실현 가능성 평가 (5분)

설계가 아무리 멋져도, 실현할 수 없으면 의미가 없습니다. 아래 체크리스트로 자신의 시나리오를 평가해보세요.

- [ ] **도구 지원**: 연동하려는 도구가 MCP를 지원하는가? (또는 API가 있는가?)
- [ ] **접근 권한**: 입력 데이터에 접근 권한이 있는가? (예: 해당 Google Sheets를 볼 수 있는가?)
- [ ] **품질 검증**: 출력 결과의 품질을 어떻게 검증할 것인가? (누가 최종 확인하는가?)
- [ ] **폴백 계획**: 자동화가 실패했을 때 수동으로 대체할 수 있는가?

4개 모두 체크할 수 있다면 실현 가능성이 높은 시나리오입니다. 체크할 수 없는 항목이 있다면, 해당 항목이 "구현 시 해결해야 할 과제"가 됩니다. 템플릿에 메모해두세요.

> **이 시나리오의 실제 구현은 Season 2에서 다룹니다.** 지금은 "설계도"를 그리는 단계입니다. 좋은 설계도가 있으면 구현은 훨씬 수월해집니다.

### 공유 타임 (5분)

2~3명이 자신의 자동화 시나리오를 간단히 발표합니다.

발표 포맷:
- **어떤 업무**를 자동화 대상으로 선택했는가? (10초)
- **현재 몇 분**이 걸리고, 자동화하면 **몇 분**으로 줄어드는가? (10초)
- **어떤 도구 연동**이 필요한가? (10초)
- 실현 가능성 평가에서 **가장 걸리는 항목**은? (20초)

다른 참가자의 시나리오를 들으면서, 자신의 업무에도 적용할 수 있는 아이디어가 있는지 찾아보세요.
