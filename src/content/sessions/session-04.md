### 세션 개요

이번 세션에서는 세션 3에서 배운 Context Engineering 개념을 바탕으로, 자신의 업무에 최적화된 **about-me.md**와 **working-rules.md**를 직접 작성합니다. 작성한 컨텍스트를 실제 Claude에 적용하여 세션 2의 프롬프트 결과가 어떻게 개선되는지 체험합니다.

### 학습 목표

- 자신의 역할, 업무, 클라이언트 정보를 about-me.md로 구조화하여 작성할 수 있다
- AI의 행동 규칙, 보고서 형식, 금지사항을 working-rules.md로 명문화할 수 있다
- 컨텍스트 파일 적용 전후의 결과물 차이를 직접 비교하고 설명할 수 있다

### 미션 브리핑 (5분)

이번 미션은 세미나 전체에서 **가장 중요한 증거 구간**입니다. 여기서 작성하는 about-me.md와 working-rules.md는 단순 연습이 아니라, 여러분이 실제 업무에 AI를 활용할 준비가 되었다는 것을 보여주는 **핵심 산출물**입니다.

세션 2에서 "AI가 생각보다 일반적인 답변을 한다"고 느꼈다면, 오늘 그 이유를 직접 해결합니다. AI에게 여러분이 누구인지, 어떤 규칙으로 일하는지 알려주는 문서를 만들면, **같은 프롬프트로도 완전히 다른 수준의 결과**를 받을 수 있습니다.

> 오늘 만드는 산출물은 경영진 보고 자료에 포함됩니다. 실제 업무에 바로 쓸 수 있는 수준으로 작성해주세요.

### 실습 가이드 (20분)

#### Step 1: about-me.md 작성 (8분)

about-me.md는 AI에게 "나는 이런 사람입니다"를 알려주는 자기소개서입니다. 아래 작성 예시를 참고하여 자신의 버전을 작성하세요.

**작성 예시 (미디어 바이어)**

```markdown
# About Me

- 나는 미디어 바이어로 디지털광고팀에서 일합니다
- 주로 퍼포먼스 광고 매체 구매 및 최적화를 담당합니다
- 주요 매체: Meta Ads, Google Ads, 네이버 검색광고
- 주요 클라이언트: A뷰티(화장품), B푸드(식음료)
- 매주 매체별 성과를 분석하고 예산 재배분을 결정합니다
- 선호하는 결과물: 간결한 표 형식, 수치 중심, 액션 아이템 포함
```

**작성 템플릿 (빈칸을 채워주세요)**

```markdown
# About Me

- 나는 _____(직책)으로 _____(부서/팀)에서 일합니다
- 주로 _____을(를) 담당합니다
- 주요 도구/매체/플랫폼: _____
- 주요 클라이언트/프로젝트: _____
- 주기적으로 반복하는 핵심 업무: _____
- 선호하는 결과물 형식: _____
```

작성 팁:
- 6줄 이상 쓰려고 하지 마세요. 핵심만 담으면 됩니다.
- AI가 추론할 수 없는 정보를 우선으로 넣으세요 (클라이언트명, 내부 용어 등).
- "나는"으로 시작하면 쓰기 쉽습니다.

#### Step 2: working-rules.md 작성 (7분)

working-rules.md는 AI의 행동 규칙서입니다. "이렇게 해라, 이건 하지 마라"를 명확하게 정의합니다.

**작성 예시 (미디어 바이어)**

```markdown
# Working Rules

## 반드시 지켜야 할 것
- 모든 수치에는 출처를 명시하세요 (GA, Meta Ads Manager 등)
- 불확실한 정보는 추측하지 말고 "확인 필요"로 표시하세요
- 결과물은 반드시 한국어로 작성하세요
- 금액은 원화(₩)로, 천 단위 구분 쉼표를 사용하세요

## 보고서 형식
- 제목 → 핵심 요약 (3줄) → 상세 데이터 (표) → 액션 아이템
- 표에는 반드시 전주 대비 증감률(%)을 포함하세요

## 하지 말아야 할 것
- 실제 클라이언트 예산 금액을 외부에 노출하지 마세요
- 경쟁사를 비하하는 표현을 사용하지 마세요
- "아마도", "것 같습니다" 같은 모호한 표현을 피하세요
```

**작성 템플릿 (빈칸을 채워주세요)**

```markdown
# Working Rules

## 반드시 지켜야 할 것
- _____
- _____
- _____

## 보고서/결과물 형식
- _____
- _____

## 하지 말아야 할 것
- _____
- _____
- _____
```

작성 팁:
- "반드시"와 "하지 마"를 구분하면 AI가 규칙을 더 정확하게 따릅니다.
- 과거에 AI 결과물에서 불만족스러웠던 부분을 떠올려보세요. 그것이 곧 규칙이 됩니다.
- 3개씩만 먼저 쓰고, 나중에 추가하면 됩니다. 완벽할 필요 없습니다.

#### Step 3: 컨텍스트 테스트 (5분)

작성한 about-me.md와 working-rules.md가 실제로 효과가 있는지 직접 확인합니다.

1. **세션 2에서 만든 프롬프트를 준비합니다** (미션 #1에서 작성한 업무 요청 프롬프트)
2. **Claude 새 대화를 열고**, 먼저 about-me.md와 working-rules.md 내용을 붙여넣은 뒤 같은 프롬프트를 입력합니다
3. **Before/After를 비교합니다**: 세션 2의 결과물 vs 오늘 컨텍스트를 추가한 결과물

비교 포인트:
- 결과물이 내 업무 맥락에 맞게 구체적으로 바뀌었는가?
- 내가 정한 형식과 규칙을 따르고 있는가?
- 실무에 바로 사용할 수 있는 수준인가?

### 공유 타임 (5분)

2-3명이 발표합니다. 발표 질문:

> "내가 만든 working-rules.md 중 가장 중요한 규칙은 무엇이고, 왜 그 규칙을 넣었나요?"

다른 참가자의 규칙을 듣고 자신의 working-rules.md에 추가할 아이디어를 얻어보세요. 같은 팀이라도 역할에 따라 규칙이 다를 수 있습니다.

### 증거 수집 안내

이번 세션의 산출물은 세미나 효과를 측정하는 핵심 데이터입니다.

- **사전/사후 설문**: 세션 시작 전 / 종료 후 설문 링크가 공유됩니다. 반드시 둘 다 완료해주세요.
- **산출물 제출**: about-me.md와 working-rules.md를 지정된 Google Drive 폴더에 업로드합니다.
- **참석률 기록**: 이 시점의 참석률과 산출물 제출율이 경영진 보고에 사용됩니다.

> 제출된 산출물은 개인 평가가 아닌, 세미나 프로그램의 효과를 입증하는 데 사용됩니다. 부담 없이, 하지만 성실하게 작성해주세요.

### 과제

다음 세션(세션 5) 전까지 아래 과제를 완료해주세요:

1. **about-me.md 보강**: 오늘 작성한 내용에 빠진 정보가 없는지 검토하고, 필요하면 클라이언트 정보나 업무 도구를 추가하세요.
2. **working-rules.md 실전 테스트**: 이번 주 실제 업무 중 AI에게 요청할 일이 생기면, working-rules.md를 함께 제공하고 결과를 관찰하세요. 규칙이 잘 지켜지지 않는 항목이 있으면 표현을 수정하세요.
3. **Before/After 스크린샷 저장**: 컨텍스트 적용 전후의 결과물 차이를 스크린샷으로 캡처해두세요. 이후 세션에서 활용합니다.
