### 세션 개요

세션 5에서 Skill의 개념과 구조를 배웠습니다. 이번 세션에서는 직접 자신의 반복 업무 1개를 분석하고, 이를 SKILL.md 형식으로 설계합니다. 완성된 Skill은 실제로 AI에게 전달하여 실행할 수 있는 수준이어야 합니다.

### 학습 목표

- 자신의 반복 업무를 입력(Inputs) / 프로세스(Process) / 주의사항(Gotchas)으로 분해할 수 있다
- SKILL.md 형식에 맞춰 실행 가능한 Skill 문서를 작성할 수 있다
- 동료 검증을 통해 Skill의 완성도를 점검하고 개선할 수 있다

---

### 미션 브리핑 (5분)

**미션**: 여러분이 매주 반복하는 업무 1개를 골라 SKILL.md로 만드세요.

완성 기준은 간단합니다: **이 문서만 읽고 다른 사람(또는 AI)이 같은 결과물을 만들 수 있어야 합니다.**

세션 5에서 본 경쟁사 브리프, 회의록 정리, 미디어 플랜 초안 예시를 참고하되, 여러분만의 실제 업무로 만들어야 합니다.

**진행 순서:**
1. 반복 업무 분석 (5분) — 업무를 분해하는 워크시트 작성
2. SKILL.md 작성 (10분) — 분석 내용을 Skill 형식으로 변환
3. 동료 교차 검증 (5분) — 옆 사람과 서로 검토

---

### 실습 가이드 (20분)

#### Step 1: 반복 업무 분석 (5분)

Skill을 바로 작성하기 전에, 먼저 업무를 분해해봅니다. 아래 워크시트를 채워보세요.

```markdown
## 나의 반복 업무 분석

**업무명**: [여기에 입력]
**빈도**: 매일 / 매주 / 매월
**현재 소요 시간**: 약 [X]시간

### 이 업무의 흐름을 단계별로 적어보세요
1. [첫 번째 단계 — 무엇을 하는가?]
2. [두 번째 단계]
3. [세 번째 단계]
4. [네 번째 단계]
5. [결과물은 무엇인가?]

### 매번 달라지는 것 (입력 = Inputs)
- [예: 이번 주 날짜, 클라이언트명, 최신 데이터]

### 항상 같은 것 (프로세스 = Process)
- [예: 보고서 형식, 분석 항목, 정렬 기준]

### 자주 틀리거나 빠뜨리는 것 (주의사항 = Gotchas)
- [예: 전주 대비 증감률 빠뜨림, 단위 오류]
```

> **Tip**: "이 업무를 내일부터 후배에게 넘겨야 한다면?" 이라고 상상하면서 적으면 훨씬 구체적으로 쓸 수 있습니다.

#### Step 2: SKILL.md 작성 (10분)

Step 1에서 분석한 내용을 아래 템플릿에 옮겨 적으세요. 각 섹션의 안내를 참고하여 작성합니다.

```markdown
---
name: [스킬 이름 — 동사로 시작: "주간 보고서 생성", "회의록 정리"]
description: [이 스킬이 무엇을 하는지, 언제 사용하는지 — AI가 이 설명을 보고 실행 여부를 판단합니다]
---

# Objective
[이 스킬의 최종 목표를 한 문장으로 — "매주 월요일 경쟁사 광고 활동 브리프를 생성한다"]

# Inputs
[매번 달라지는 정보 — Step 1의 "매번 달라지는 것"에서 가져옴]
- [입력 1]: [설명과 예시]
- [입력 2]: [설명과 예시]

# Process
[단계별 실행 순서 — Step 1의 흐름에서 가져옴. 가능한 구체적으로.]
1. [단계 1 — 무엇을 하는지, 어떤 데이터를 참조하는지]
2. [단계 2]
3. [단계 3]
4. [결과물 형식과 저장 위치]

# Output Format
[결과물의 구조 — 제목, 섹션, 표 형식 등]

# Gotchas
[Step 1의 "자주 틀리거나 빠뜨리는 것"에서 가져옴]
- [주의사항 1]
- [주의사항 2]
- 불확실한 정보는 추측하지 말고 "확인 필요"로 표시할 것
```

> **작성 시 체크 포인트:**
> - name은 동사로 시작하면 용도가 명확해집니다 ("보고서" 보다 "주간 보고서 생성")
> - description은 AI가 "이 스킬을 실행할까 말까"를 판단하는 기준이 됩니다. 트리거 상황을 포함하세요
> - Process의 각 단계는 "왜?"가 아니라 "무엇을 어떻게?"에 집중하세요
> - Gotchas에 최소 2개 이상의 주의사항을 포함하세요. 실무에서 가장 가치 있는 섹션입니다

#### Step 3: 동료 교차 검증 (5분)

옆 사람과 짝을 이루어 서로의 SKILL.md를 교환하세요. 다음 두 가지 질문을 던져보세요:

1. **"이 설명만 보고 같은 작업을 할 수 있겠어?"**
   - 빠진 단계가 있다면 어디에서 막히는지 구체적으로 알려주세요
   - "이 단계에서 뭘 봐야 하는지 모르겠어"처럼 피드백하면 좋습니다

2. **"빠진 단계나 예외 상황은 없어?"**
   - 본인은 당연하게 여기지만 다른 사람은 모르는 것이 반드시 있습니다
   - "만약 ~한 경우에는 어떻게 해?" 같은 질문을 던져보세요

검증 결과를 반영하여 SKILL.md를 수정하세요.

---

### 공유 타임 (5분)

2~3명이 자신의 Skill을 간단히 발표합니다.

발표 포맷:
- **어떤 업무**를 Skill로 만들었는가? (10초)
- 가장 어려웠던 부분은? (20초)
- 동료 검증에서 어떤 피드백을 받았는가? (20초)

---

### 품질 체크리스트

제출 전에 아래 항목을 모두 확인하세요:

- [ ] 다른 사람이 이 Skill만 보고 같은 결과물을 만들 수 있는가?
- [ ] 입력(Inputs)이 명확하게 정의되어 있는가? (예시 포함)
- [ ] 프로세스가 빠짐없이 순서대로 기술되어 있는가?
- [ ] 예외 상황/주의사항(Gotchas)이 포함되어 있는가? (최소 2개)
- [ ] 결과물 형식(Output Format)이 구체적으로 명시되어 있는가?

---

### 제출 안내

완성된 SKILL.md 파일을 다음과 같이 제출합니다:
- 파일명: `SKILL-[본인이름].md` (예: `SKILL-김수현.md`)
- 제출 방법: 공유 폴더에 업로드 또는 강사에게 직접 전달

---

### 과제

이번 주 안에 다음 2가지를 완료하세요:

1. **Skill 실행 테스트**: 오늘 만든 SKILL.md를 실제로 AI에게 전달하고 실행해보세요. 결과물이 기대와 다른 부분이 있다면 Skill을 수정하세요.
2. **Skill 1개 추가 작성**: 오늘 만든 것 외에 다른 반복 업무 1개를 추가로 SKILL.md로 작성해보세요. 두 번째는 훨씬 빠르게 작성할 수 있을 것입니다.
