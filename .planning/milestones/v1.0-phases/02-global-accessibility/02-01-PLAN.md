Make two changes to `src/layouts/BaseLayout.astro`:

**Change 1 — Add skip link as first child of `<body>`, before `<nav>`:**

Insert the following `<a>` element immediately after the `<body>` opening tag (currently line 28) and before the `<nav>` element (currently line 29):

```html
<a
  href="#main-content"
  class="sr-only focus:not-sr-only focus:fixed focus:top-4 focus:left-4 focus:z-[60] focus:bg-accent focus:text-bg focus:font-mono focus:font-semibold focus:text-sm focus:px-4 focus:py-2 focus:rounded-btn"
>
  본문으로 바로가기
</a>
```

Key details:
- `sr-only` hides the link visually but keeps it in the tab order and accessibility tree
- `focus:not-sr-only` reveals it when focused via keyboard Tab
- `focus:z-[60]` places it above the sticky nav (which is `z-50`)
- `focus:bg-accent focus:text-bg` matches the primary CTA button pattern
- `focus:font-mono focus:font-semibold focus:text-sm` matches existing button typography
- `focus:rounded-btn` uses the 2px border-radius token
- Text is Korean `본문으로 바로가기` per the all-Korean UI convention

Do NOT use `display:none` or `visibility:hidden` — these remove the element from tab order.
Do NOT add `tabindex` to the `<main>` element — scroll-to is sufficient.

**Change 2 — Add `id="main-content"` to the `<main>` element:**

The current `<main>` tag (line 58) is:
```html
<main class="max-w-5xl mx-auto px-4 py-8" role="main">
```

Change it to:
```html
<main id="main-content" class="max-w-5xl mx-auto px-4 py-8" role="main">
```

This makes the skip link's `href="#main-content"` target functional.
