## Directory Purposes

**`src/components/`:**
- Purpose: Reusable Astro components shared across pages
- Contains: `.astro` files only (no framework components)
- Key files:
  - `SessionCard.astro` -- card UI for session listings on curriculum page
  - `AdoptionLadder.astro` -- 5-step progress indicator used on index + about pages

**`src/content/sessions/`:**
- Purpose: Markdown content for all seminar sessions
- Contains: 13 `.md` files with Zod-validated frontmatter + Markdown body
- Naming: `session-{number}.md` where number is zero-padded 2 digits (01-12), plus `session-bonus.md`
- The `session-bonus.md` file has `week: 0` and `number: 0`, which excludes it from the regular curriculum grid (curriculum page only renders weeks 1-6)

**`src/layouts/`:**
- Purpose: Page-level layout wrappers
- Contains: Single `BaseLayout.astro` file
- Provides: HTML document structure, sticky nav with mobile menu, footer, external font loading, global CSS import

**`src/pages/`:**
- Purpose: File-based routing -- each `.astro` file becomes a URL
- Contains: 7 static pages + 1 dynamic route directory
- `src/pages/sessions/[...slug].astro` uses Astro rest params for dynamic session detail pages

**`src/styles/`:**
- Purpose: Global CSS and design token definitions
- Contains: Single `global.css` with Tailwind v4 `@import "tailwindcss"` and `@theme` block
- All color, font, and border-radius tokens are defined here

**`public/`:**
- Purpose: Static assets served at site root without processing
- Contains: Favicons only (templates directory exists but is empty)
- Files here are accessible at `{base}/filename`

**`.github/workflows/`:**
- Purpose: CI/CD automation
- Contains: `deploy.yml` -- builds Astro site and deploys to GitHub Pages on push to main

**`.planning/`:**
- Purpose: Planning and analysis documents for AI-assisted development
- Contains: Codebase mapping documents
- Committed: Yes

**`.gstack/`:**
- Purpose: Design review and QA report artifacts generated by gstack tools
- Contains: Screenshots, audit reports, design baselines
- Committed: No (gitignored)

## Key File Locations

**Entry Points:**
- `astro.config.mjs`: Astro build configuration (site URL, base path, Vite plugins)
- `src/pages/index.astro`: Landing page / site homepage
- `.github/workflows/deploy.yml`: CI/CD pipeline entry

**Configuration:**
- `astro.config.mjs`: Framework config -- site `https://elymas.github.io`, base `/agent-seminar`
- `tsconfig.json`: TypeScript strict mode extending `astro/tsconfigs/strict`
- `package.json`: Dependencies (astro, tailwindcss, @tailwindcss/vite), scripts (dev, build, preview)
- `src/content.config.ts`: Content Collection schema with Zod validation
- `src/styles/global.css`: Design token definitions via Tailwind v4 `@theme`

**Core Logic:**
- `src/pages/sessions/[...slug].astro`: Most complex page -- dynamic routing, content rendering, TOC generation, prev/next navigation
- `src/pages/curriculum.astro`: Data fetching, sorting, and grouping of sessions by week
- `src/content.config.ts`: Content schema that validates all session frontmatter

**Testing:**
- No test files exist in this project. No test framework is configured.

## Naming Conventions

**Files:**
- Pages: `kebab-case.astro` (e.g., `season2.astro`, `curriculum.astro`)
- Components: `PascalCase.astro` (e.g., `SessionCard.astro`, `AdoptionLadder.astro`)
- Layouts: `PascalCase.astro` (e.g., `BaseLayout.astro`)
- Content: `kebab-case.md` with numeric prefix (e.g., `session-01.md`)
- Config files: `kebab-case` (e.g., `content.config.ts`, `astro.config.mjs`)

**Directories:**
- All lowercase, no hyphens: `components`, `layouts`, `pages`, `content`, `sessions`, `styles`

**CSS Classes:**
- Tailwind utility classes exclusively -- no custom class names except `session-content` (used as a scoping selector for Markdown-rendered content styles in `[...slug].astro`)

**Component Props:**
- TypeScript `interface Props` defined in frontmatter
- camelCase property names matching content schema fields

## Where to Add New Code

**New Static Page:**
- Create: `src/pages/{page-name}.astro`
- Wrap content in `<BaseLayout title="...">`
- Include the base URL normalization pattern at the top of the frontmatter:
  ```typescript
  const rawBase = import.meta.env.BASE_URL;
  const base = rawBase.endsWith('/') ? rawBase : rawBase + '/';
  ```
- Add navigation link in `BaseLayout.astro` (both desktop nav and mobile menu sections)

**New Reusable Component:**
- Create: `src/components/PascalName.astro`
- Define `interface Props` in the frontmatter block
- Use Tailwind utility classes with theme tokens (e.g., `bg-bg-card`, `text-accent`, `font-mono`)
- Import in page files: `import PascalName from '../components/PascalName.astro';`

**New Content Session:**
- Create: `src/content/sessions/session-{NN}.md`
- Include all required frontmatter fields per the Zod schema in `src/content.config.ts`:
  - Required: `number`, `title`, `week`, `weekTheme`, `type`, `level`, `keyMessage`, `deliverable`, `timeStructure`
  - Optional: `mission`, `bookRef`, `status` (defaults to `upcoming`)
- The session will automatically appear on the curriculum page if `week` is 1-6
- A detail page route is auto-generated from the file name (slug = `session-{NN}`)

**New Design Token:**
- Add to the `@theme` block in `src/styles/global.css`
- Follow the existing naming: `--color-{semantic-name}`, `--font-{family}`, `--radius-{element}`
- Use in templates as Tailwind classes: `bg-{token-name}`, `text-{token-name}`

**New Layout:**
- Create: `src/layouts/LayoutName.astro`
- Include `<slot />` for content injection
- Import global CSS: `import '../styles/global.css';`

**New Static Asset:**
- Place in: `public/` directory
- Reference with base-path prefix: `{base}filename.ext`

## Special Directories

**`dist/`:**
- Purpose: Astro build output (static HTML, CSS, JS)
- Generated: Yes, by `npm run build`
- Committed: No (gitignored)

**`.astro/`:**
- Purpose: Astro-generated type definitions and internal cache
- Generated: Yes, automatically by Astro
- Committed: No (gitignored)

**`node_modules/`:**
- Purpose: npm dependencies
- Generated: Yes, by `npm install`
- Committed: No (gitignored)

**`.gstack/`:**
- Purpose: gstack tool artifacts (design audits, QA reports, screenshots)
- Generated: Yes, by gstack design-review and qa tools
- Committed: No (gitignored)

**`public/templates/`:**
- Purpose: Reserved for downloadable template files
- Generated: No
- Committed: Yes, but currently empty
- Note: Templates are currently displayed as inline `<pre>` blocks on `resources.astro` rather than as downloadable files
