## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Skip-to-content link: `px-4 py-2` (16px × 8px) — matches existing button padding convention
- Focus ring offset: 2px — matches `--radius-btn` visual rhythm
- Minimum touch target for menu-toggle button: 44px × 44px (existing `p-2` on a 20px icon gives ~36px; no change required — keyboard fix only, touch target is out of scope for this phase)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0a` (`--color-bg`) | Page background, body surface |
| Secondary (30%) | `#111827` (`--color-bg-card`) | Cards, nav background, footer |
| Accent (10%) | `#10b981` (`--color-accent`) | See reserved list below |
| Destructive | n/a | No destructive actions in this phase |

Accent (`#10b981`) reserved for:
1. Focus-visible ring on ALL interactive elements — 2px solid ring, 2px offset, on `:focus-visible` only
2. Skip-to-content link background (visible state) — consistent with existing primary CTA pattern
3. Skip-to-content link text is `--color-bg` (#0a0a0a) — dark text on accent fill

Focus ring specification:
- `outline: 2px solid var(--color-accent)`
- `outline-offset: 2px`
- Applied via `:focus-visible` pseudo-class only (not `:focus`, to avoid mouse-click outlines)
- Must NOT use `outline: none` anywhere without a replacement
- Ring color contrast against dark background (#0a0a0a): #10b981 is WCAG AA compliant (contrast ratio ~5.4:1 against #0a0a0a)

> Source: CLAUDE.md Design System section, global.css tokens, REQUIREMENTS.md EXP-03

## Interactive Element Inventory

All elements that MUST receive the `:focus-visible` ring treatment:

| Element | Location | Current Focus State |
|---------|----------|---------------------|
| Site logo link | BaseLayout.astro nav | None |
| Nav links (소개, 커리큘럼, 일정, 자료실, FAQ) | BaseLayout.astro nav | None |
| Mobile menu toggle button | BaseLayout.astro nav | None |
| Mobile nav links (same 5) | BaseLayout.astro mobile-menu | None |
| Session cards (`<a>` wrapping `<article>`) | SessionCard.astro | None |
| Progress bar segment links | sessions/[...slug].astro | None |
| Breadcrumb link | sessions/[...slug].astro | None |
| TOC links | sessions/[...slug].astro | None |
| Prev/Next nav links | sessions/[...slug].astro | None |
| FAQ `<details>/<summary>` toggles | faq.astro | Native (inconsistent) |
| In-session content `<a>` links | sessions/[...slug].astro `.session-content` | None |

Implementation approach: Single global CSS rule in `global.css`:
```css
*:focus-visible {
  outline: 2px solid var(--color-accent);
  outline-offset: 2px;
}
```
This covers all current and future interactive elements without per-component changes.

## Skip-to-Content Link

### DOM Position
First child of `<body>` in BaseLayout.astro — before the `<nav>` element.

### Visible State (on focus)
- Position: fixed, top-left corner
- Background: `var(--color-accent)` (#10b981)
- Text color: `var(--color-bg)` (#0a0a0a)
- Padding: `px-4 py-2` (16px × 8px)
- Font: `font-mono font-semibold text-sm`
- Border-radius: `rounded-btn` (2px)
- z-index: above nav (nav is z-50, skip link uses z-[60])

### Hidden State (not focused)
- Moved off-screen: `sr-only` pattern (not `display:none` — must remain focusable)
- CSS: `absolute w-px h-px p-0 -m-px overflow-hidden whitespace-nowrap border-0`

### Target
- `href="#main-content"` — links to `<main>` element
- `<main>` must have `id="main-content"` added (currently has `role="main"` but no id)

### Copywriting
- Link text: `본문으로 바로가기` (Korean: "Skip to main content")
- Matches site's all-Korean UI convention
