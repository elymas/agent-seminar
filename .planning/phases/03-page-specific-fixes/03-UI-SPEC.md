### EXP-02: FAQ Contact Block (faq.astro)

Augment the existing contact card (lines 171–176) with an actionable contact mechanism. The contact value is a required user-supplied input that the plan task must surface before execution.

| Element | Copy |
|---------|------|
| Existing line 1 (unchanged) | `위에 없는 질문이 있으신가요?` |
| Existing line 2 (unchanged) | `세미나 운영팀에 문의하시면 답변을 드리겠습니다.` |
| New contact link | `[CONTACT_EMAIL]` rendered as `<a href="mailto:[CONTACT_EMAIL]">` |

Visual treatment for the contact link: `font-mono text-accent hover:text-accent-hover text-xs transition-colors` — accent color, mono font, consistent with existing CTA link patterns.

If the actual contact email is unavailable at execution time, the plan task must insert a clearly marked placeholder: `[CONTACT_EMAIL]` in a code comment, flagged for manual replacement. Do not fabricate an email address.

**Slack alternative pattern** (if email is unavailable): `<p class="text-text-muted text-xs mt-2">Slack <span class="font-mono text-text-secondary">#seminar-questions</span> 채널에 남겨주세요.</p>` — this option requires no `href` and is safe to use as a placeholder without an actual URL.

**Source:** RESEARCH.md EXP-02 pattern. Contact value unknown — flagged as open question.

### EXP-05: Session Progress Bar Accessible Labels ([...slug].astro)

Add `aria-label` to each `<a>` in the progress bar `.map()` expression. The `title` attribute is retained as a tooltip fallback. No copy changes to visible text.

| Element | Copy |
|---------|------|
| Non-current session link | `Session {n}으로 이동` |
| Current session link | `Session {n} (현재 세션)으로 이동` |

The Korean suffix `으로 이동` ("navigate to") is appended to each label. The `(현재 세션)` parenthetical appears only on the active segment (where `n === number`). The `<html lang="ko">` attribute set in `BaseLayout.astro` ensures Korean in `aria-label` is correctly announced by screen readers.

**Source:** RESEARCH.md EXP-05 pattern + WCAG 2.1 SC 4.1.2 / ARIA14.
